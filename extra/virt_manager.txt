Virt-manager - это опенсорсный гуй для виртуальных машин на базе KVM (и наверное других тоже).

Ставим так:
    sudo pacman -S virt-manager dnsmasq swtpm qemu-full
    # dnsmasq нужен для сети
    # swtmp нужен для эмуляции TPM, который нужен для win11
    # qemu-full нужен для поддержки всяких дополнительных фичей, например для выбора GPU = virtio
    sudo systemctl enable libvirtd
    sudo systemctl start libvirtd

Если ругается на сеть, попробуй `sudo virsh net-start default`.

Для винды:
    Все настройки в гуи дефолтные.
    В устройствах проверь что есть TPM.
Для линукса:
    Обязательно при создании машины нажми "проверить настройки перед установкой", и там поменяй BIOS на UEFI, а то потом никак нельзя.
    Hardware acceleration для GPU.
        На Nvidia что-то пока не работает. Я находил всякие хаки типа этого, но не пробовал:
            https://gitlab.com/libvirt/libvirt/-/issues/311#note_2587889800
        На AMD должно сработать так:
            1. В `Видео Virtio`: по дефолту галочка в неопределенном значении, непонятно, включено по дефолту или нет. Стоит включить.
            2. В `Дисплей Spice`:
                   Включи OpenGL (если это не сделать, а сделать только пункт (1), то не дает начать установку и ругается на это)
                   Поставь `тип ожидания = нет` (иначе под галочкой opengl он пишет ворнинг про это)
                   Выбери свою видеокарту в списке

При установке винды, чтобы не просил аккаунт, на экране "для личного использования или для работы" открой консоль через shift+f10, напиши `ncpa.cpl` и там выключи сеть.
Если вдруг прозевал и он требует создать аккаунт, то в той же консоли `oobe\bypassnro` перезапустит эту программу настройки ПК, и сможешь попробовать еще раз.
